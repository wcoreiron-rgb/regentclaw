Zero Trust Principles

Six Pillars of Governed Security

RegentClaw operationalizes zero trust beyond network segmentation — enforcing governance at every layer of the security stack.

🪪
Every component has identity
Agents, claws, workflows, and service accounts are first-class identity principals. Non-human identities are governed alongside human ones.
🔐
Every action is authorized
No security action executes without passing policy evaluation, risk scoring, and (when required) human approval via the Trust Fabric pipeline.
👁️
Every runtime is monitored
Behavioral baselines for all claws and agents surface anomalies in real time, before policies are even invoked.
🧾
Every workflow is attributable
The immutable audit log captures the full decision chain — who triggered what, which policies matched, what risk score was assigned, and what happened.
🚧
Every risk is containable
Emergency mode provides instant containment actions. Each claw can isolate its scope without requiring full platform access.
⚙️
Every module is governed
All 24 claws share the same governance layer — there are no escape hatches, no unmonitored paths, and no ungoverned execution modes.
Quick Start

Up and Running in Minutes

RegentClaw is fully containerized. A single docker compose up starts the entire stack — backend API, frontend, PostgreSQL, and Redis.

Seed scripts populate the database with example policies, agents, connectors, and workflows so you can explore the platform immediately.

Read Full Documentation →
🐚 bash 
# Clone and start
git clone https://github.com/your-org/regentclaw
cd regentclaw
docker compose up -d

# Seed the database (run once)
python seed_policies.py && \
python seed_agents.py

# Access the platform
# Frontend:  http://localhost:3000
# API:       http://localhost:8000
# API Docs:  http://localhost:8000/docs
